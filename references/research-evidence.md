# 研究与证据协议

## 素材充分性检查（写作简报后、研究前执行）

在进入正式研究之前，先评估已有素材的充分性。这不是"充足/不足"的二值判断，而是分维度评估并决定后续动作。

### 检查维度

```yaml
material_readiness:
  topic: sufficient          # 主题是否明确
  audience: sufficient       # 目标读者是否明确
  core_opinion: partial      # 是否有初步观点方向
  evidence: insufficient     # 是否有足够外部证据
  personal_experience: missing # 是否有第一手经历
  voice_context: sufficient  # 是否有文风上下文
  allowed_output: outline    # 基于充分性决定的输出上限
  required_actions:          # 进入下一阶段前必须完成的动作
    - research
    - add_experience_anchor
```

### 输出级别与动作映射

| 评估结果 | allowed_output | required_actions |
|---|---|---|
| 全部 sufficient | draft | 无 |
| evidence 不足 | outline | research |
| personal_experience 缺失 | draft（带编辑锚点） | add_experience_anchor |
| core_opinion 缺失 | outline | find_core |
| topic 不明确 | — | clarify_with_user |
| 全部缺失 | — | clarify_with_user |

### 降级策略

当 WebSearch 不可用或外部证据不足时：
1. 模型先验知识可作为研究线索或暂定背景，但不得直接作为事实使用；
2. 时间敏感信息（数字、日期、引语、产品状态）必须核验，不得依赖先验知识；
3. 无法核验的信息显式标记为 `unverified_background`，不得作为关键事实；
4. 对缺失的第一手经历设置编辑锚点；
5. 告知用户哪些部分需要补充外部来源；
6. 不因信息不足而编造事实。

## 研究问题优先

先写问题，再寻找答案。至少覆盖：

1. 这个主题当前可确认的事实是什么？
2. 常见解释是什么？
3. 哪些证据支持拟议观点？
4. 哪些证据反对或限制它？
5. 有哪些第一手来源？
6. 哪些信息可能已经过时？
7. 哪个具体案例最能承载文章价值？

## 排斥分析

在回答研究问题后，执行排斥分析以发现非显然角度：

1. **谁被现有方案/解释排除在外？** — 现有方案通常服务于谁，忽略了谁？
2. **被排除的人怎样解决自己的问题？** — 他们的替代方案是什么？
3. **这个替代方案揭示了什么被忽略的需求？** — 这是最强角度的来源。

排斥分析的角度优先于"大众角度"——大众角度意味着没有信息增量。

## 痛点深度检查

对研究发现的痛点进行三层深度检查：

- **Layer 1（问题存在）**：问题是什么？— 最浅层，大多数文章停在这里。
- **Layer 2（后果）**：这个问题导致了什么具体后果？— 有数据或案例支撑。
- **Layer 3（荒谬应对行为）**：人们为了应对这个问题，采取了什么荒谬的应对行为？— 最强层，但**不得编造**。没有 Layer 3 素材时，标注编辑锚点，不要虚构。

## 材料分类

- `F` Fact：可外部核验的事实。
- `O` Opinion：作者或来源的判断。
- `I` Inference：由事实推出、但尚非事实的推测。
- `E` Experience：作者确认的第一手经历。

正文可以隐藏标签，但工作材料必须保留分类。

## 来源等级

- A：官方文件、原始数据、论文、当事人原话。
- B：可信媒体、专业机构、完整访谈。
- C：社区讨论、二手总结、未系统验证的个案。
- D：无法定位原始出处的转述，仅可以作为线索。

关键论断不得只依赖 C/D。引用二手来源时尽可能回溯原始来源。

## 证据密度

不是每段都要塞数据。主要章节至少应具备一个有效价值载体：

- 数据；
- 具体案例；
- 原始引述；
- 可验证机制；
- 作者确认的经历；
- 清晰的因果链。

## 研究停止条件

研究不能无限扩张。停止条件分为三层：必须满足、尽量满足、无法满足时退出。

### 必须满足（全部满足才能结束研究）

1. **关键事实可追溯**：Core Statement 的关键事实已有可靠证据（来源等级 A 或 B）。
2. **主要论断有证据**：每个主要论断至少有一个价值载体（数据/案例/原始引述/可验证机制/作者经历/因果链）。
3. **未解决项有标记**：无法核验的信息已标记，缺失经历已设置编辑锚点。

### 尽量满足（满足越多越好，但不阻塞退出）

4. **找到反方或边界**：至少找到一个有力反方或边界。对于个人经历型文章，如果外部反方证据不存在，可以用作者基于真实经历进行自我质疑代替。
5. **主要章节有价值载体**：每个章节都已挂载证据。
6. **连续研究无重要增量**：新增来源连续两轮没有产生重要新信息。
7. **来源时效满足要求**：关键数据未过时。

### 无法满足时的退出状态

如果必须满足的条件中有无法达成的，不要无限研究，应输出退出状态：

| 无法满足的条件 | 退出状态 | 说明 |
|----------------|----------|------|
| 关键事实无法追溯 | `evidence_conflict` | 证据相互冲突或来源不可靠 |
| 作者第一手经历缺失 | `needs_author_input` | 需要作者补充经历后才能继续 |
| 研究已尽力但证据仍不足 | `research_limited` | 已收集可用证据，但不足以支撑完整文章，降级输出大纲 |
| 核心观点在研究中崩塌 | `core_broken` | 研究发现与核心观点根本矛盾 |

**关键原则**：不要为了满足"尽量满足"条件而无限研究。个人经历型文章可能天然没有外部反方证据，这不应阻塞研究结束。

## 禁止行为

- 为了完整而补造数字；
- 把相关性写成因果；
- 用单个评论代表"大家"；
- 把产品宣传语当客观结论；
- 引用不存在或未读过的来源；
- 用模糊的"研究表明"逃避出处。
