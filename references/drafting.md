# 初稿协议

## 写作顺序

1. 写出每节的最小论证骨架。
2. 把证据和案例放到对应位置。
3. 补充解释和过渡。
4. 对照 Voice Profile 做基础表达调整。
5. 标记缺失的作者材料。

## 基本原则

- 具体先于抽象，展示先于解释。
- 观点必须回答证据，而不是漂浮在证据上方。
- 每段只推进一个小动作。
- 过渡用于保持主线，不用于装饰流畅。
- 引用和数据服务论证，不炫耀研究量。
- 结尾回收核心或推动行动，不必强行升华。

## 编辑锚点

统一格式：

```markdown
[编辑锚点/E：请补充你在该场景中的真实动作、感受或原话]
[编辑锚点/O：请确认你是否愿意明确表达这个判断]
[编辑锚点/F：该事实尚缺原始来源，发布前必须核验]
```

编辑锚点不得伪装成正文，不得在“完成稿”中静默删除。

## 写作与去 AI 味的边界

初稿只做必要的可读性调整，不运行全面禁词、句长扰动、碎句治理或口语化重写。内容审稿通过后，再交给专门的去 AI 味 Skill。
