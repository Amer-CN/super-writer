# 找核与攻核

## 两层选题评分

在进入找核七问之前，先用两层评分判断选题是否值得继续。

### 第一层：HKR 阅读价值（每项 0-5 分，共 15 分）

- **H — 好奇**：是否制造合理的信息缺口？读者看到标题会产生"想知道"的冲动吗？
- **K — 知识**：是否提供信息增量？读者看完会比看之前多知道一些有用的东西吗？
- **R — 共鸣**：是否与目标读者的真实处境相关？读者会觉得"这说的是我"吗？

### 第二层：UEPT 可写性门禁（每项 0-5 分，共 20 分）

- **U — Unique**：是否有独特角度？和已有文章相比有没有不同之处？
- **E — Evidence**：是否有足够证据？能找到数据、案例、原始来源吗？
- **P — Personal**：作者是否有真实经验或判断资格？能提供第一手视角吗？
- **T — Timely**：是否存在时效窗口？现在写比三个月后写更好吗？

### 门禁规则

- HKR 至少两项达到 3 分，否则建议放弃或重构选题。
- U < 2：先找角度，不急着写。
- E < 2：先研究，素材不够不要硬写。
- P < 2：禁止虚构第一人称经验，改用第三人称或分析视角。
- H/K/R 都低：建议放弃或重构选题。

### 输出格式

```yaml
topic:
  appeal:
    H: 4
    K: 5
    R: 3
  feasibility:
    U: 2
    E: 1
    P: 4
    T: 3
  decision: research_before_writing  # write / research_before_writing / find_angle / abandon
  diagnosis:
    - "有知识价值，但目前角度不独特"
    - "作者有第一手经验"
    - "外部证据不足"
```

不输出单一总分，输出画像和诊断。

## 找核七问

1. 大多数文章会怎样讲这个题目？
2. 读者已经知道什么？
3. 哪个事实、冲突或结果最反常？
4. 现有解释遗漏了什么？
5. 作者真正相信什么？
6. 文章愿意反对什么？
7. 读者看完后，哪一个认知应该发生变化？

## 四铲深化

七问找到方向后，用四把铲子挖到支撑角度的底层逻辑。可组合使用，直到挖到能抵抗质疑的想法。

1. **反转铲**：说反面。如果反面荒谬，说明原观点太浅（停留在常识层）。如果反面也成立，说明发现了真正的张力。
   - 示例：原观点"AI 工具提升效率"→ 反转"AI 工具降低效率"→ 如果反转也成立（效率提升的代价是判断力外包），就找到了更深的观点。

2. **追问前提铲**："这是对的，但只有当 ___ 也对。"找出隐含前提，检验它是否真的成立。
   - 示例："远程办公提高满意度"→ 前提是"员工有独立工作的能力和资源"→ 检验这个前提。

3. **追问情绪铲**：为什么这个话题让人不舒服或兴奋？情绪指向的地方通常藏着真正的利益冲突。
   - 示例：讨论"35岁危机"让人焦虑→焦虑指向的是"年龄与能力的不对等关系"→这才是真正的话题。

4. **翻转定义铲**：常见词的真正含义是什么？重新定义一个常见概念，可能打开全新角度。
   - 示例："效率"通常理解为"单位时间产出"→翻转为"单位注意力产出"→整个论证框架变了。

## 核心卡

- **Core Statement**：一句可被反驳、有证据支持的判断。
- **Reader Change**：读者阅读前后的认知差异。
- **Core Tension**：两股相互冲突但同时成立的力量。
- **Value Carrier**：最值得记住或转发的单一元素。
- **Scope**：结论适用于什么、不适用于什么。

## 攻核六刀

1. **反例刀**：最明显的反例是什么？
2. **因果刀**：是否存在更合理的替代因果？
3. **边界刀**：在哪些人群、时间或场景下不成立？
4. **常识刀**：这是否只是常识换了新说法？
5. **概括刀**：是否从一个案例跨越到了普遍结论？
6. **钢人刀**：最聪明的反对者会怎样完整反驳？

## 结论

- `holds`：核心成立，可以进入结构。
- `morphs`：核心需变形，更新核心卡后再测试。
- `breaks`：核心崩塌，返回研究或停止。

禁止用"这个观点很有启发"作为压力测试结果。
